# 請求先サブアカウント IAM権限最適化 要件定義書

## 1. 目的と背景

2026年3月末展開のGoogle Cloudの仕様変更に伴い、顧客（サブアカウントの利用者）に付与している権限を最小権限の原則（Least Privilege）に基づき適正化する。
現状の `roles/billing.admin` は過剰な権限であるためこれを剥奪し、利便性を損なわない必要十分な権限セットに置き換える。

## 2. 対象スコープ

- **対象リソース**: 親請求先アカウント（`${PARENT_ACCOUNT_ID}`）に紐づく全ての請求先サブアカウント。
- **対象ユーザー**: IAMポリシー内で `user:*` として登録されているメンバー。
- **除外対象**: 自社ドメイン（`${YOUR_DOMAIN}`）を持つユーザー（運用管理者）は変更対象から除外する。

## 3. 権限変更内容

対象ユーザーに対して以下のIAMポリシー変更を実施する。

- **剥奪するロール**:
  - `roles/billing.admin` (請求先アカウント管理者)
- **付与するロール**:
  - `roles/billing.user` (請求先アカウントユーザー)
  - `roles/billing.viewer` (請求先アカウント閲覧者)
  - `roles/billing.costsManager` (請求先アカウント費用管理者)

## 4. 実行要件と非機能要件

- **実行環境**: Google Cloud Shell (認証済み状態)
- **安全性 (Dry-Run)**: 誤操作を防ぐため、デフォルトはDry-Run（対象のリストアップのみ）として動作し、明示的な `--apply` フラグを付与した場合のみ実際の変更を行う設計とする。
- **監査ログ**: 実行時の標準出力をタイムスタンプ付きのログファイルとしてローカルに保存し、監査証跡として残す。
- **耐障害性**: 個別のIAM変更API呼び出しが失敗した場合でも、スクリプト全体を異常終了させず、エラーをログに記録して次のユーザーの処理を継続する。
